## 简介

数据科学是一门将原始数据转化为理解、洞见与知识的学科。

《R for Data Science》这本书的目标，是帮助大家掌握 R 语言中最重要的工具，让你能够高效、可复现地开展数据科学工作。

## 你将学到什么

数据科学是一个广阔的领域，你不可能仅凭一本书就掌握全部内容。本书旨在为你打下最重要工具的坚实基础，并让你掌握足够的知识，以便在需要时能找到更多学习资源。我们对典型数据科学项目流程的梳理大致如图 1 所示。

<img src="./images/base.png" width="600" />

> **图 1** 在典型的数据分析流程中，首先需要**导入数据**并**完成数据清洗**。然后通过**数据转换、可视化与建模**的循环迭代来理解数据。最后，将结果**分享**给他人。

首先，必须将数据**导入 R**。这通常是指把存储在文件、数据库或网络应用程序接口（API）中的数据，加载到 R 里的 data-frame 中。如果无法将数据导入 R，就无法对它进行分析！

导入数据后，最好对其进行清洗（tidy）。**数据清洗**是指将数据以一种统一的格式存储，让数据的存储形式与其语义保持一致。简单来说，**清洗数据**就是：**每一列代表一个变量，每一行代表一次观测**。整理数据之所以重要，是因为这种统一的结构能让你把精力集中在**解决数据问题**上，而不是耗费时间把数据强行整理成不同函数所需的格式。

有了整洁的数据，接下来通常要对其进行**转换**。转换包括：

- 筛选出你感兴趣的观测变量（比如某个城市的所有人、过去一年的所有数据）
- 根据已有变量创建新变量（比如利用距离和时间计算速度）
- 计算一系列汇总统计量（比如计数或均值）

**清洗与转换合在一起称为数据整理（wrangling）**，因为把数据变成便于使用的格式，往往真的像一场 “战斗”！

有了包含所需变量的整洁数据，接下来是**两大产生知识的核心工具**：**可视化**与**建模**。它们各有所长、也各有不足，因此任何真实的数据分析都会在二者之间**反复迭代**。

**可视化本质上是为了让人很容易理解。**优秀的可视化能向你展示意料之外的信息，或是引发关于数据的新问题。好的可视化还可能提示你：你提出的问题是否恰当，或是需要收集不同类型的数据。可视化可以带来惊喜，但它的扩展性并不算好，因为它需要人类进行解读。

**模型**是可视化的互补工具。当你把问题定义得足够精确，就可以用模型来回答这些问题。模型本质上是数学工具，通常具有很好的扩展性。即便扩展性不足，通常购买更多计算机也比雇佣更多人更划算！但所有模型都基于假设，**模型无法质疑自身的假设**。这意味着模型从根本上无法给你带来意外发现。

数据科学的最后一步是**交流**，这是任何数据分析项目中**关键**的一环。除非你能将结果传达给他人，否则无论你的模型和可视化多么出色、让你多么深刻地理解了数据，都毫无意义。

所有这些工具的基础是**编程**。编程是一项**贯穿始终的工具**，你几乎在数据科学项目的每一个环节都会用到。要成为一名成功的数据科学家，你不必成为编程专家，但学习更多编程知识会让你受益匪浅，因为优秀的编程能力能让你自动化处理常规任务，并更轻松地解决新问题。

你会在每个数据科学项目中用到这些工具，但对大多数项目而言，它们还不够。这里大致遵循**80/20 法则**：用本书学到的工具，你可以处理每个项目中约 80% 的工作，但剩下的 20% 则需要借助其他工具。在整本书中，我们都会为你指明可以进一步学习更多知识的资源。

## 本书的组织结构

前面介绍的数据科学工具，大致按照你在分析中使用它们的顺序来安排的。但根据我们的经验，先学习数据导入与整理并不是最优选择，因为这类工作 80% 的时候既常规又枯燥，剩下 20% 又棘手又让人沮丧。对于一门新学科来说，这可不是一个好的起点！因此，我们会从已经导入并整理好的数据开始，先学习可视化与数据转换。这样一来，在你后续自己去导入和整理数据时，动力会一直保持高涨，因为你知道这些付出是值得的。

在每一章中，我们都尽量遵循统一的结构：先用一些启发性的示例让你看清整体框架，然后再深入细节。本书的每个部分都配有练习题，帮助你练习所学内容。尽管跳过练习题很有诱惑力，但没有什么比通过解决实际问题来学习更有效的了

## 本书不包含的内容

本书不会覆盖所有重要的主题。我们认为，**专注于核心内容**非常重要，这样你才能尽可能快地入门并动手实践。这也意味着本书无法涵盖所有重要主题。

**建模**

建模在数据科学中极为重要，但这是一个庞大的主题，本书篇幅有限，无法对其进行全面讲解。想要学习更多建模知识，我们强烈推荐同事马克斯・库恩（Max Kuhn）与朱莉娅・西尔格（Julia Silge）所著的《R 语言整洁建模》（[Tidy Modeling with R](https://www.tmwr.org/)）。这本书教你使用 **tidymodels** 系列包。从名字你或许就能猜到，它与本书所用的 **tidyverse** 包遵循许多相同的设计规范。

**大数据**

本书**主要聚焦于小型内存数据集**。这是一个合适的起点，因为如果你没有处理小数据的经验，就无法应对大数据。本书大部分内容所教的工具，可以轻松处理**数百兆字节**的数据；稍加注意，通常也能用来处理 GB 级别的数据。我们还会介绍如何从数据库和 Parquet 文件中获取数据，这两者常被用于存储大数据。你不一定能处理整个数据集，但这不成问题，因为你只需要**子集或抽样数据**，就可以解答你关心的问题。

如果你经常处理更大的数据（比如 10–100 GB），那么建议你深入学习 [data.table](https://github.com/Rdatatable/data.table)。本书不会讲解它，因为它与 tidyverse 使用不同的接口，且需要你学习一些不一样的规则。不过，它的运行速度极快，如果你需要处理大数据，投入时间学习它所带来的性能提升是值得的。

**Python, Julia 及其它语言**

在本书中，你不会学到关于 Python、Julia 或其他任何适用于数据科学的编程语言。这并不是因为我们认为这些工具不好 —— 它们**非常优秀**！在实际工作中，大多数数据科学团队都会混合使用多种语言，通常至少会用到 R 和 Python。但我们坚信，**最好一次只精通一种工具**，而 R 就是一个极佳的起点。

## 学习前提

为了让你充分掌握本书内容，我们对你已有的知识做了一些基本假设。你需要具备基本的数理常识，若已有一些**编程基础**会更有帮助。如果你从未接触过编程，我们推荐 Garrett 所著的[Hands On Programming with R](https://rstudio-education.github.io/hopr/)，作为本书的补充读物非常有价值。

要运行本书中的代码，你需要准备四样东西：R 语言环境、RStudio 编辑器、tidyverse 包集合，以及少量其他辅助包。包（Package）是可复现的 R 代码的基本单元，它包含可复用的函数、说明函数用法的文档，以及示例数据。

### R

从 CRAN 下载 R 语言，官网地址为：https://cloud.r-project.org。R 语言每年会推出一个全新的主版本，且每年会发布 2-3 个次版本，建议你定期更新版本。版本升级可能会稍显繁琐，尤其是主版本升级时，需要重新安装所有的 R 包，但拖延升级只会让后续的操作更麻烦。本书推荐使用 R 4.2.0 及以上版本。

### RStudio

RStudio 是一款适用于 R 语言编程的集成开发环境（IDE），你可以从官网地址https://posit.co/download/rstudio-desktop/ 下载该软件。RStudio 每年会更新数次，有新版本推出时软件会自动提醒，无需手动查看更新。建议你定期升级，以使用其最新、最实用的功能。阅读本书，需确保你的 RStudio 版本至少为 2022.02.0。

### tidyverse

你还需要安装一些 R 包。**R 包**是函数、数据和文档的集合，能够扩展基础 R 的功能。使用包是成功运用 R 语言的关键。本书中你将要学习的大部分包，都属于 **tidyverse**。tidyverse 的所有包都遵循共同的数据理念与 R 编程思想，并且被设计成可以协同工作。

你可以用一行代码安装:

```R
install.packages("tidyverse")
```

在你的电脑上，将这行代码输入到控制台中，然后按下回车键运行。R 会从 CRAN（综合 R 语言档案网络）下载这些包，并将它们安装到你的电脑上。

在使用 `library()` 加载包之前，你无法使用包中的函数、对象或帮助文件。安装完包之后，你可以通过 `library()` 函数来加载它：


tidyverse 的包更新频率相当高。可以运行 `tidyverse_update()` 函数来查看是否有可用的更新。

### 其它包

还有许多非常优秀的包并不属于 tidyverse，因为它们解决的是不同领域的问题，或是基于另一套底层设计理念开发的。这并不代表它们更好或更差，只是**定位不同**而已。换句话说，与 tidyverse 互补的并不是 “混乱的工具集”，而是许多相互关联的其他工具生态。当你用 R 完成更多数据科学项目时，你会学到新的包，以及新的数据分析思维方式。

本书中我们还会使用许多 **tidyverse 以外**的包。例如，我们会用到下面这些包，因为它们提供了有趣的数据集，方便我们在学习 R 语言的过程中进行练习：

```R
install.packages(
  c("arrow", "babynames", "curl", "duckdb", "gapminder", 
    "ggrepel", "ggridges", "ggthemes", "hexbin", "janitor", "Lahman", 
    "leaflet", "maps", "nycflights13", "openxlsx", "palmerpenguins", 
    "repurrrsive", "tidymodels", "writexl")
  )
```

我们还会选用一些其他包来做**一次性示例**。你现在不需要安装它们，只要记住：每当你看到类似这样的报错时：

```R
library(ggrepel)
#> Error in library(ggrepel) : there is no package called ‘ggrepel’
```

你需要运行 `install.packages("ggrepel")` 来安装这个包。

## 运行 R 代码

上一节为你展示了多个运行 R 代码的示例。本书中的代码格式如下：

```R
1 + 2
#> [1] 3
```

如果你在本地控制台中运行这段相同的代码，显示效果会如下所示：

```R
> 1 + 2
[1] 3
```

主要有两点区别。在你的控制台中，你在 `>` 后面输入代码，这个符号叫作**提示符**；书中不会显示提示符。在书中，运行结果用 `#>` 标注为注释；而在控制台里，结果会直接显示在代码后面。这两点区别意味着：如果你使用的是本书的电子版，可以轻松地把代码从书中复制并粘贴到控制台运行。

在本书中，我们使用**统一约定**来指代代码：

- 函数以**等宽代码字体**显示，并在后面加上圆括号，例如 `sum()` 或 `mean()`。
- 其他 R 对象（如数据或函数参数）也使用等宽字体，但**不带圆括号**，例如 `flights` 或 `x`。
- 有时为了明确某个对象来自哪个包，我们会在包名后加上**两个冒号**，例如 `dplyr::mutate()` 或 `nycflights13::flights`。这种写法本身也是**合法有效的 R 代码**。
